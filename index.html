Key edge cases covered

• Wearables accessible independently after sign-up — no booking required
• GP slot gating (4–5 working day window with plain-English reason)
• 15-min slot hold countdown during checkout
• Payment failure recovery (slots still held)
• Slot expiry (released → pick new)
• Locked account (too many attempts)
• Empty/partial: no wearables, no reports, no appointments, incomplete onboarding
• Add-on options visible with cost context (nurse home visit £70, hormone panel £85)
• Non-blocking pre-consult tasks with skip/connect-later
• ID verification skip (defer to later)
• Fasting/preparation reminder auto-offered
• Data & Insights hub merges wearables data, device management, and medical history questionnaire
• Working Ask Evi chat with response simulation based on keyword matching
• Document hub with Evida reports + uploaded documents with rename
• Message threads with unread counts and flat thread list
• FAQ with accordion, category filtering, and search

UAT reviewer notes

Navigating the prototype: Open any screen from the grid above. Each screen is a self-contained mobile-first HTML file.

State explorer: Click the U button (bottom-right corner of any screen) to toggle the UAT toolbar on/off. The toolbar lets you switch between empty, loading, error, and edge-case states without manipulating localStorage. Once enabled, the toolbar persists across page navigation.

Keyboard shortcuts (booking flow): The 15-min slot hold timer starts at step 5. Payment has ~20% random failure rate for testing recovery.

Test accounts: locked@evida.co.uk (any password) triggers the locked-account state on the sign-in screen.

Nav note: Bottom nav labels say "Data & Insights" (merged Insights + Wearables). The standalone Wearables page is still accessible for device management. No notification centre in MVP.

Known limitations: Standalone GP consult (non-Baseline) removed from dashboard — baseline-only MVP. Multi-device wearables connect via Terra simulation. Ask Evi responses are simulated (keyword-matching, no real AI backend).
